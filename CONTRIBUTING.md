# Contributing

Thanks for your interest in contributing to `unipiazza-mcp`.

We welcome:

- bug reports
- documentation improvements
- tests
- focused code improvements

If you want to discuss a change before opening a pull request, contact `dev@unipiazza.it`.

## Ways to contribute

- Open an issue describing the bug, gap, or proposal.
- Send a pull request with a focused change and a clear explanation.
- Reach out by email if the change touches product behavior, authentication, or deployment assumptions.

## Development notes

- Edit source files in `src/`, not generated files in `build/`.
- Run `npm test` and `npm run build` before submitting a pull request.
- Keep changes small and easy to review when possible.
